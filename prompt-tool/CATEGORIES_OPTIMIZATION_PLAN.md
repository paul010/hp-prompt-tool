# 分类系统优化计划

## 当前问题分析

### 1. 分类重复问题
- **演示演讲** / **演示汇报** - 同一场景的两个名称
- **技术支持** / **IT支持** - 同一场景的两个名称
- **市场营销** / **Marketing** - 同一场景的两个名称

### 2. 提示词分布不均
| 分类 | 提示词数量 | 状态 |
|------|-----------|------|
| 客户服务 | 28个 | 过多 |
| 商务决策 | 20个 | 过多 |
| 技术开发 | 13个 | 较多 |
| 项目管理 | 7个 | 正常 |
| 办公沟通 | 10个 | 正常 |
| 技术支持 | 6个 | 偏少 |
| 数据智能 | 2个 | 过少 |
| 内容创作 | 2个 | 过少 |
| 学习成长 | 2个 | 过少 |
| 演示演讲 | 1个 | 过少 |
| 多语言翻译 | 1个 | 过少 |
| 市场营销 | 0个 | 空缺 |

### 3. UI显示问题
- 旧分类（LEGACY_SCENARIOS）在侧边栏显示，与新分类重复
- 共计20+个分类标签，用户体验不佳

## 优化方案

### 方案A：精简分类（推荐）
保留8个核心业务场景，每个场景至少5个提示词：

| 新分类名称 | 合并自 | 目标数量 |
|-----------|--------|---------|
| 办公协作 | 办公沟通 + 项目管理 | 15+ |
| 数据分析 | 数据智能 + 商务决策（分析类） | 10+ |
| 技术开发 | 技术开发 + 技术支持 | 20+ |
| 内容创作 | 内容创作 + 市场营销 | 10+ |
| 客户服务 | 客户服务（保留现有） | 15+ |
| 学习成长 | 学习成长 + 培训相关 | 10+ |
| 演示汇报 | 演示演讲 + 多语言翻译（演示相关） | 5+ |
| 商务决策 | 商务决策（战略类） | 10+ |

**优势：**
- 分类数量减半，更易选择
- 每个分类都有足够内容
- 避免重复标签

### 方案B：渐进优化
保持现有12分类，但：
1. 移除LEGACY_SCENARIOS中的重复项
2. 合并真正重复的分类
3. 补充提示词到空缺分类
4. 在UI中只显示新分类

**优势：**
- 保持更细粒度的分类
- 向后兼容性更好
- 改动较小

## 推荐实施步骤（方案A）

### Phase 1: 简化SCENARIOS配置
1. 移除LEGACY_SCENARIOS在UI中的显示
2. 合并重复分类ID
3. 更新类型定义

### Phase 2: 重组提示词分类
1. 重新分配现有提示词到8个新分类
2. 补充空缺分类的提示词

### Phase 3: 优化输入字段设计
1. 创建"Combobox"组件（可选择+可输入）
2. 为常用字段添加预设建议
3. 支持自定义输入

### Phase 4: 补充提示词
每个分类补充到至少5个提示词

## Combobox输入组件设计

```typescript
// 新的combobox字段类型
type FieldType =
  | "text"        // 纯文本
  | "combobox"    // 可选择+可输入（新增）
  | "textarea"    // 多行文本
  | "select"      // 纯选择
  | "multiselect" // 多选
  | "number"
  | "date"
  | "email"
  | "url";

// InputField配置示例
{
  name: "jobTitle",
  label: { en: "Job Title", zh: "职位" },
  type: "combobox",         // 使用combobox
  preset: "jobTitle",       // 引用预设选项
  required: true,
  placeholder: { en: "Select or type...", zh: "选择或输入..." }
}
```
