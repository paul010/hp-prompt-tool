## 二、整合后的分类体系（推荐方案）

### 核心原则
1. **扁平化**：控制在 8-10 个主要分类
2. **直观性**：用户能快速找到所需提示词
3. **扩展性**：便于后续添加新分类
4. **包容性**：涵盖企业和通用场景

### 新分类体系设计

#### 分类映射关系

| 旧分类 | 新分类 | 理由 |
|--------|--------|------|
| 办公效率 | 办公沟通 | 更聚焦沟通场景 |
| 数据分析 | 数据智能 | 更符合现代数据工作 |
| 编程开发 | 技术开发 | 更简洁明了 |
| 创意写作 | 内容创作 | 覆盖更广的创作场景 |
| 客户服务 | 客户服务 | 保持不变，加入客户成功 |
| 项目管理 | 项目管理 | 保持不变 |
| 演示汇报 → 演示演讲 | 专项能力 |
| 翻译本地化 → 多语言翻译 | 专项能力 |
| **新增** | 商务决策 | 整合高管、经理、销售等决策类场景 |
| **新增** | 市场营销 | 整合营销相关场景 |
| **新增** | 技术支持 | 整合IT支持相关 |

### 场景标签体系（二级筛选）

```
一级：业务场景（8+4=12个）
二级：角色标签（销售、产品、HR、IT、高管、工程师等）
三级：技能标签（邮件、PPT、SQL、Python、写作等）
```
